# kiri (v0.1.0)

**kiri** — минималистичный, умный локальный TUI-помощник (Terminal User Interface) для ухода за растениями. Программа плавно рассчитывает «уровни жажды», напоминает о поливе, ведёт подробный журнал ухода и динамически подстраивается под погоду и сезон для уличных растений.

Все данные хранятся исключительно локально на вашем компьютере в легковесной базе SQLite.

---

## Особенности

- **Полноценный интерактивный TUI:** вкладки, сетка карточек с динамическим цветом, календарь задач на месяц, мышь и клавиатура.
- **Динамическая модель «жажды»:** уровень воды уменьшается плавно; для уличных растений учитываются сезон, температура и осадки (Open-Meteo API).
- **Календарь и журнал:** планирование полива и хронология ухода.
- **UNIX-way и скриптуемость:** флаги `--status`, `--summary`, `--json` для скриптов и панелей (`Waybar`, `Polybar`, `i3blocks`, `tmux`).
- **Прозрачный режим (`--transparent`):** без фоновых заливок — удобно для тайловых WM и кастомных тем терминала.
- **Лёгковесность:** локальная SQLite, один бинарник без внешних runtime-зависимостей.

## Быстрый старт

### Требования

- **Go 1.22** или выше.
- Любой современный эмулятор терминала (Alacritty, Ghostty, Kitty, GNOME Terminal и др.).

### Запуск из исходников

```bash
git clone https://github.com/freislot/kiri.git
cd kiri
go run .
```

### Компиляция бинарника

```bash
CGO_ENABLED=1 go build -o kiri .
./kiri
```

> **Совет для быстрого старта:** при самом первом запуске `kiri` автоматически заполнит базу **9 демо-растениями** с разными уровнями жажды. Можно сразу проверить интерфейс, графики и календарь без ручной настройки.

Чтобы пересоздать демо-данные:

```bash
rm ~/.config/kiri/data.db
go run .
```

### Базовое управление

- `Tab` / `Shift+Tab` или `1`–`5` — переключение между вкладками
- `↑` / `↓` или `j` / `k` — навигация по спискам
- `Enter` — просмотр деталей растения
- `q` / `Ctrl+C` — выход из приложения

Полный список флагов — в разделе [Флаги запуска](#флаги-запуска).

## Установка

### Прямая загрузка (Pre-compiled Binaries)

Готовый бинарник или `.deb`-пакет — на странице **Releases**.

Для Ubuntu/Debian:

```bash
sudo apt install ./kiri_0.1.0_linux_amd64.deb
```

### Arch Linux (AUR)

> ⏳ **Примечание:** официальные пакеты в AUR (`kiri` / `kiri-bin`) появятся в ближайших обновлениях.

Пока пользователи Arch могут собрать `kiri` из исходников через `go` CLI или `Makefile`; бинарник попадёт в системный `PATH`.

### Через Go CLI

```bash
go install github.com/freislot/kiri@latest
```

### Сборка через Makefile

```bash
make              # скомпилировать бинарник
sudo make install # установить бинарник и man-страницу
sudo make uninstall # удалить из системы
```

## Пути к файлам и XDG-стандарт

Приложение следует спецификации **XDG Base Directory**. Данные изолированы в домашней директории:

| Файл | Путь |
|------|------|
| **База данных SQLite** | `~/.config/kiri/data.db` |
| **Конфигурация TOML** | `~/.config/kiri/settings.toml` |
| **Резервные копии** | `~/.config/kiri/data.db.bak` / `settings.toml.bak` |

> **Примечание:** коэффициенты высыхания, пороговые температуры, объёмы осадков и сезонные множители настраиваются в `settings.toml`, пока приложение **закрыто**. Подробное описание математической модели полива — в системном руководстве: `man kiri` (после `make install` или установки `.deb`).

Файл `settings.toml` можно править вручную (язык, город, интервал погоды, секция `[model]` и т.д.).

### Заставка при запуске

При старте показывается заставка **kiri** — ASCII-логотип на фоне лёгкого анимированного тумана (символы `. ~ *`).

| Фаза | Длительность | Что происходит |
|------|--------------|----------------|
| Проявление | ~2 с | Туман на экране, логотип построчно «печатается» из дымки |
| Пауза | ~1 с | Полный логотип на фоне тумана |
| Переход | ~0,7 с | Плавный переход к основному интерфейсу |

**Enter** — пропустить заставку сразу. **q** / **Ctrl+C** — выход.

Заставку можно отключить навсегда: **Настройки → Быстрый запуск** (см. ниже).

## Флаги запуска

Справка по флагам: `kiri -h`

| Флаг | Описание |
|------|----------|
| `--version` | Показать версию программы (`v0.1.0`) и выйти. База не открывается, TUI не запускается |
| `--status` | Вывести форматированную таблицу статуса растений в консоль и выйти (без TUI). Язык — из `settings.toml` |
| `--summary` | Вывести одну строку для статус-баров (например, `💧 kiri: 3 требуют полива!` или `🌿 kiri: ok`) и выйти |
| `--json` | Вывести полный машиночитаемый JSON-дамп всех растений для интеграций и выйти |
| `--transparent` | Принудительно запустить TUI в прозрачном режиме (без фоновых заливок); настройка сохраняется в `settings.toml` |

Флаги можно комбинировать с обычным запуском, но **`--version`**, **`--status`**, **`--summary`**, **`--json`** и интерактивный режим друг друга исключают: срабатывает только один режим (в порядке проверки: `--version` → `--status` → `--summary` → `--json` → TUI).

### `--version`

```bash
./kiri --version
# kiri v0.1.0
```

### `--status`

Краткий отчёт для скриптов, cron и быстрой проверки без открытия интерфейса:

```bash
./kiri --status
```

Пример вывода (русский интерфейс):

```
РАСТЕНИЕ           СТАТУС         ПОЛИВ
Флоренция (улица)  [████░░░░░░]   Через 2 дня
Ромашка (комната)  [██████████]   Уже полито
Мята (улица)       [░░░░░░░░░░]   СРОЧНО!
```

Колонки:

- **Растение** — название и тип (`улица` / `комната` или `outdoor` / `indoor`);
- **Статус** — полоска жажды из 10 символов (`█` / `░`);
- **Полив** — `СРОЧНО!`, `Уже полито` (уровень ≥ 80%, порог не в `[model]`) или `Через N дней` / `In N days`.

Перед выводом учитывается высыхание с момента последнего обновления (как в TUI), но база **не изменяется**. Автобэкап при `--status` **не выполняется**.

### `--summary`

Одна строка для виджетов, статус-бара и быстрой проверки:

```bash
./kiri --summary
```

Примеры:

```
🌿 kiri: ok
💧 kiri: 3 требуют полива!
```

Если хотя бы одно растение требует полива (`СРОЧНО!` в `--status`), выводится строка с 💧 и числом. Иначе — 🌿 `ok`. Логика та же, что у `--status`; база не изменяется.

### `--json`

Машиночитаемый дамп всех растений для скриптов и интеграций:

```bash
./kiri --json
```

Пример (сокращённо):

```json
[
  {
    "id": 1,
    "name": "Мята",
    "location": "garden",
    "base_interval_days": 7,
    "is_outdoor": true,
    "water_level": 0,
    "state": "water_now",
    "needs_watering": true,
    "next_action": "WATER NOW!",
    "watering_due_date": "2026-06-28",
    "days_until_watering": 0
  }
]
```

Поля `needs_watering`, `watering_due_date` и `days_until_watering` вычисляются так же, как в `--status`. Поле **`next_action`** — текстовая подсказка модели на английском (`Water soon`, `Water tomorrow`, `Water in N days`, `WATER NOW!` и т.д.); в TUI **не используется**, только в JSON и в колонке БД. Высыхание учитывается в памяти, база не изменяется.

### `--transparent`

```bash
./kiri --transparent
```

Эквивалентно включению **Настройки → Прозрачный режим**. Подробнее — в разделе [Прозрачный режим](#прозрачный-режим-transparent-mode).

## Общий интерфейс

```
┌─────────────────────────────────────────────────────────┐
│  Растения  Календарь  Журнал  Настройки  О программе   │  ← вкладки; справа город и погода
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    основная область                     │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  Статусное сообщение…                          (пусто)  │  ← строка статуса
│  h/j/k/l навигация │ w полить │ s сдвинуть │ …         │  ← подсказки клавиш
└─────────────────────────────────────────────────────────┘
```

- **Шапка** — вкладки слева, справа название выбранного города + погода.
- **Основная область** — содержимое выбранной вкладки.
- **Строка статуса** — сообщения о действиях (полив, отмена, ошибки).
- **Футер** — горячие клавиши для текущего экрана; клавиши выделены зелёным, описания — приглушённо.

Неактивные карточки растений отображаются приглушённо; выбранная — с яркой рамкой.

### Переключение вкладок

| Клавиша | Действие |
|---------|----------|
| `Tab` / `Shift+Tab` | Следующая вкладка |
| `1` | Все растения |
| `2` | Календарь |
| `3` | Журнал ухода |
| `4` | Настройки |
| `5` | О программе |

## Вкладка «Все растения»

Главный экран — сетка карточек растений (до 3 столбцов, горизонтальный скролл при большем количестве).

### Что показывает карточка

- иконка и название;
- локация (комната, балкон и т.д.);
- полоска жажды `▰▰▰▱▱` и процент;
- строка с 📅 — **дата** ближайшего полива (или срочный статус).

**Текст на карточке** (локализован, зависит от состояния):

| Состояние | Пример (RU) |
|-----------|-------------|
| Обычное | `📅 Полив: 05.07` |
| Срочно (уровень ≤ 0) | `📅 ПОЛИТЬ СЕГОДНЯ!` |
| После лёгкого дождя | `📅 Полив: 08.07 (дождь)` |

Дата считается из текущего уровня воды и скорости высыхания (см. [Как работает «жажда»](#как-работает-жажда)). Отдельные текстовые подсказки вроде «Water soon» / «Water in N days» **на карточке не показываются** — они попадают только в поле `next_action` в базе и в вывод `--json` (см. ниже).

**Цвет полоски:** зелёный (≥ 80%), жёлтый (30–79%), красный (< 30%).

**Иконки на карточке:**

| Иконка | Значение |
|--------|----------|
| 🌿 | Всё в порядке |
| 💧 | Нужно полить срочно |
| 🌧 | Полив сдвинут из-за дождя |

Выбранная карточка подсвечивается зелёной рамкой. Срочные растения — красной.

### Навигация по сетке

| Клавиша | Действие |
|---------|----------|
| `h` / `←` | Предыдущий столбец |
| `l` / `→` | Следующий столбец |
| `k` / `↑` | Вверх внутри столбца |
| `j` / `↓` | Вниз внутри столбца |

Под сеткой — индикатор горизонтальной прокрутки (`◀ [████░░░░] ▶`), если столбцов больше, чем видно.

### Блок «Детали»

Под карточками (с отступом) — панель с подробностями о **выбранном** растении:

- базовый интервал полива;
- текущий статус (тоже **по дате** полива, не по `next_action`);
- последние записи из журнала;
- подсказка, если полив откладывали несколько раз подряд (увеличьте интервал через **изменение** растения).

### Действия с растением

| Клавиша | Действие |
|---------|----------|
| `w` | Полить (сбросить жажду до 100%) |
| `s` | Отложить полив (+postpone_boost_percent к уровню, по умолчанию +15%) |
| `a` | Добавить новое растение |
| `e` | Изменить выбранное растение |
| `d` | Удалить выбранное растение |

**Добавление растения** — всплывающая форма:

- название, локация, интервал полива в днях (1–365, по умолчанию 7);
- флаг «улица» (растение реагирует на осадки и сезон);
- `Tab` / `↑` `↓` — переключение полей;
- `Space` — вкл/выкл «улица»;
- `Enter` — сохранить;
- `Esc` — отмена.

**Изменение растения** (`e`) — та же форма, поля заполнены текущими значениями:

- можно изменить название, локацию, **интервал полива** и флаг «улица»;
- уровень жажды и история поливов сохраняются;
- если интервал **увеличили**, счётчик отложений подряд сбрасывается.

**Удаление** — запрос подтверждения:

- `y` / `Enter` — удалить;
- `n` / `Esc` — отмена.

## Вкладка «Календарь»

Экран планирования: слева — календарь месяца, справа — задачи.

Дата полива рассчитывается из текущего уровня воды и скорости «высыхания» (см. раздел [Как работает «жажда»](#как-работает-жажда)).

### Календарная сетка

- неделя начинается с **воскресенья**;
- в ячейке дня — число и индикатор:
  - `💧` — в этот день запланирован полив;
  - `⚠️` — просроченный / критический полив;
- выбранный день подсвечивается;
- сегодняшний день выделен отдельно.

### Навигация по календарю

| Клавиша | Действие |
|---------|----------|
| `h` / `←` | Предыдущий день |
| `l` / `→` | Следующий день |
| `k` / `↑` | На неделю назад |
| `j` / `↓` | На неделю вперёд |
| `Enter` / `Space` | Перейти к списку задач выбранного дня |
| `Esc` | Вернуться к календарю из списка задач |

### Сайдбар задач

**Задачи выбранного дня** — список с чекбоксами:

```
[ ] Полить Монстеру
[x] Полить Фикус
```

| Клавиша | Действие |
|---------|----------|
| `j` / `↓` | Следующая задача |
| `k` / `↑` | Предыдущая задача |
| `Space` | Отметить выполненной / снять отметку |

При отметке задачи выполненной растение **поливается до 100%**, событие записывается в журнал.
Снять галочку можно только для задач, закрытых через календарь. Если растение уже поливали вручную (`w`) в этот день, задача остаётся закрытой.

## Вкладка «Журнал ухода»

Хронологический список всех событий:

- поливы;
- отложенные поливы;
- реакции на погоду;
- добавление, изменение и удаление растений;
- выполненные задачи из календаря.

Формат даты в списке: **день + месяц + время** (например, `27 июн 14:35`).

| Клавиша | Действие |
|---------|----------|
| `j` / `↓` | Прокрутка вниз |
| `k` / `↑` | Прокрутка вверх |
| `PgDn` | Страница вниз |
| `PgUp` | Страница вверх |

## Вкладка «Настройки»

| Раздел | Описание | Управление |
|--------|----------|------------|
| **Язык** | Русский / English | `h`, `l` или `←` `→` |
| **Город** | Город для погоды (Open-Meteo) | `Enter` / `Space` / `c` — поиск; ввод текста, `↑` `↓` — выбор из списка |
| **Обновление погоды** | Интервал запроса API в минутах (**1–1440**, по умолчанию 10) | `h`, `l` или `←` `→` — шаг **±1**; в `settings.toml` можно задать любое значение в диапазоне |
| **Интервал по умолчанию** | Базовый цикл полива для **новых** растений (1–365 дней) | `h`, `l` или `←` `→` |
| **Температура-заглушка** | °C, когда нет живой погоды; пишется в `[model].fallback_temp_c` | `h`, `l` или `←` `→` |
| **Автобэкап** | Копия базы и `settings.toml` при каждом запуске | `a` или `Enter` на строке |
| **Прозрачный режим** | Без фонов интерфейса (для тайловых WM) | `t` или `Enter` |
| **Быстрый запуск** | Пропускать заставку при старте | `Enter` на строке + `h`/`l` |
| **База данных** | Путь к файлу с данными | только просмотр |
| **Бэкап** | Путь к резервной копии; ручной бэкап — `b` | `b` → подтверждение |

Все переключатели сохраняются в `settings.toml` и восстанавливаются при следующем запуске.

### Прозрачный режим (Transparent Mode)

Для пользователей **тайловых оконных менеджеров** и кастомных цветовых схем терминала.

В прозрачном режиме kiri **не рисует фоны** — остаются только текст, рамки и акцентные цвета. Терминал и обои/тема WM просвечивают сквозь интерфейс.

Включение:

```bash
./kiri --transparent
```

Или в настройках: **Прозрачный режим → Да** (`t`).

Рекомендуется также включить прозрачность в настройках самого терминала (Kitty, Alacritty, Ghostty и т.д.).

## Как работает «жажда»

У каждого растения есть **уровень воды** от 100% до 0%. Он уменьшается **плавно со временем** — пропорционально прошедшим часам, а не «скачком раз в сутки».

### Базовая формула высыхания

Скорость потери воды за сутки:

```
ΔV = (100 / интервал_полива) × K_сезон × K_темп
```

| Параметр | Смысл |
|----------|--------|
| **100 / интервал** | При идеальных условиях бак опустошается ровно за заданное число дней (например, интервал 10 → −10% в день) |
| **K_сезон** | Множитель сезона (только **уличные** растения; комнатные всегда 1.0). Значения — в `[model.season]` |
| **K_темп** | Множитель температуры воздуха. Пороги — в `[model.temp]` |

Пересчёт выполняется автоматически:

- при запуске приложения (догоняет пропущенное время);
- при действиях с растением (полив / отложить);
- в открытом приложении — фоновая проверка **примерно раз в минуту** (бар и дата полива обновляются без ваших действий).

При переходе через полночь каждый фрагмент дня считается со **своим** K_сезон (если растение на улице).

Когда уровень падает до **0%** — растение требует полива **сегодня** (статус «WATER NOW!» / 💧).

### K_сезон (только уличные растения)

Коэффициенты задаются в `[model.season]`. По умолчанию:

| Сезон | Месяцы | Поле TOML | K_сезон | Эффект |
|-------|--------|-----------|---------|--------|
| Зима | дек – фев | `winter` | **0.5** | Медленное высыхание, покой |
| Весна | мар – май | `spring` | **1.0** | Базовая линия (эталон для дождя) |
| Лето | июн – авг | `summer` | **1.5** | Быстрое высыхание, жара |
| Осень | сен – ноя | `autumn` | **0.75** | Прохладно, влага держится дольше |

Комнатные растения: **K_сезон = 1.0** круглый год.

### K_темп (температура воздуха)

Пороги и множители — в `[model.temp]`. Формула:

```
если T ≤ cool_threshold_c  →  K_темп = cool_factor
если T ≤ hot_threshold_c   →  K_темп = 1.0
если T > hot_threshold_c   →  K_темп = 1.0 + (T − hot_threshold_c) × hot_slope
```

По умолчанию (`cool_threshold_c = 20`, `cool_factor = 0.8`, `hot_threshold_c = 27`, `hot_slope = 0.05`):

| Температура | K_темп |
|-------------|--------|
| **≤ 20°C** | **0.8** — прохладно, сохнет медленнее |
| **21–27°C** | **1.0** — норма |
| **> 27°C** | **1.0 + (T − 27) × 0.05** — жара ускоряет высыхание (30°C → 1.15, 37°C → 1.5) |

**Где используется реальная температура, а где нет:**

| Сценарий | Температура |
|----------|-------------|
| Постепенное высыхание между запусками / в фоне | `[model].fallback_temp_c` (по умолчанию **22°C**) |
| Дата полива на карточке, в календаре и в деталях | `[model].fallback_temp_c` |
| Авто-учёт дождя (добавление воды при лёгком дожде) | **текущая** температура из погоды |

### Статусы растения

| Статус | Когда |
|--------|-------|
| `normal` | Уровень > 0, всё штатно |
| `water_now` | Уровень ≤ 0 — полить сегодня |
| `shifted_by_rain` | Уличное растение — вчера был лёгкий дождь; на карточке показана **дата сдвинутого полива** 🌧 |
| `new` | Интервал не задан (0 дней) |

### Поле `next_action` (модель, не TUI)

При каждом пересчёте состояния модель записывает текстовую подсказку в поле **`next_action`** (колонка SQLite, поле в `--json`). Это **не** то, что вы видите на карточке в TUI.

| Уровень / расчёт | Значение `next_action` (EN) |
|------------------|----------------------------|
| Уровень ≤ `water_soon_percent` (по умолчанию 20%), но > 0 | `Water soon` |
| До полива ≤ 1 день | `Water tomorrow` |
| Иначе | `Water in N days` |
| Уровень ≤ 0 | `WATER NOW!` |
| Интервал не задан | `Configure interval` |

**На карточке TUI** вместо этих строк всегда показывается **дата** (`Полив: ДД.ММ`) или срочный статус — см. [Что показывает карточка](#что-показывает-карточку).

Дата полива считается так:

```
дней_до_полива = ⌈уровень_воды / ΔV⌉
```

где ΔV — суточная скорость высыхания при `[model].fallback_temp_c`.

### Ручные действия

| Действие | Эффект |
|----------|--------|
| **Полить** (`w`) | Уровень → 100%, сброс счётчика отложений, статус `normal` |
| **Отложить** (`s`) | **+postpone_boost_percent** к уровню (по умолчанию **+15%**, не больше 100%); после **postpone_suggest_after** отложений подряд (по умолчанию **2**) — рекомендация увеличить базовый интервал на 1 день (применить через `e`) |

## Параметры модели полива (`[model]` в settings.toml)

Все коэффициенты «жажды» вынесены в секцию **`[model]`** файла `~/.config/kiri/settings.toml`. Их можно менять **без пересборки** — правьте файл, пока kiri **не запущен**. Нулевые или некорректные значения автоматически заменяются на встроенные defaults.

Пример (значения по умолчанию):

```toml
[model]
fallback_temp_c = 22.0
water_soon_percent = 20
postpone_boost_percent = 15
postpone_suggest_after = 2

[model.season]
winter = 0.5
spring = 1.0
summer = 1.5
autumn = 0.75

[model.rain]
light_mm = 1.0
heavy_mm = 10.0
base_shift_hours = 24.0

[model.temp]
cool_threshold_c = 20
cool_factor = 0.8
hot_threshold_c = 27
hot_slope = 0.05
```

### Базовая формула (напоминание)

Суточная скорость потери воды для растения с интервалом полива `N` дней:

```
V_base = 100 / N
ΔV     = V_base × K_сезон × K_темп     (% воды в сутки)
```

При высыхании за произвольный интервал времени:

```
уровень_новый = max(0, уровень − ΔV × (часы / 24))
```

Каждый календарный день считается **со своим** K_сезон (если растение на улице). K_сезон для комнатных растений всегда **1.0**.

---

### `[model]` — общие параметры

| Поле | По умолчанию | Назначение |
|------|--------------|------------|
| **`fallback_temp_c`** | `22.0` | Температура (°C), когда **нет** живой погоды: фоновое высыхание, расчёт **даты** полива на карточке и в календаре. Допустимый диапазон: 1–40. Редактируется в TUI: **Настройки → Температура-заглушка**. |
| **`water_soon_percent`** | `20` | Порог для `next_action = "Water soon"` (уровень ≤ этого %, но > 0). **В TUI не отображается** — на карточке по-прежнему дата полива; жёлтый цвет полоски жажды задаётся отдельно (30–79%). |
| **`postpone_boost_percent`** | `15` | Сколько **процентов** добавляется к уровню воды при отложении полива (`s`). Итог ограничен 100%. |
| **`postpone_suggest_after`** | `2` | После скольких **подряд** отложений показать совет увеличить базовый интервал на +1 день. |

---

### `[model.season]` — K_сезон по времени года

| Поле | По умолчанию | Месяцы | Смысл |
|------|--------------|--------|-------|
| **`winter`** | `0.5` | дек – фев | Зимний покой: вода уходит медленнее |
| **`spring`** | `1.0` | мар – май | Эталон; от него считается сдвиг от дождя |
| **`summer`** | `1.5` | июн – авг | Жара и испарение: вода уходит быстрее |
| **`autumn`** | `0.75` | сен – ноя | Прохлада: влага держится дольше |

**Формула:** для уличного растения в данный календарный день

```
K_сезон = коэффициент_месяца     (winter / spring / summer / autumn)
```

Для комнатных: `K_сезон = 1.0` всегда.

Чем **больше** K_сезон, тем **быстрее** падает уровень воды.

---

### `[model.temp]` — K_темп от температуры воздуха

| Поле | По умолчанию | Смысл |
|------|--------------|-------|
| **`cool_threshold_c`** | `20` | Верхняя граница «прохладного» диапазона (°C) |
| **`cool_factor`** | `0.8` | K_темп при T ≤ cool_threshold_c (высыхание медленнее) |
| **`hot_threshold_c`** | `27` | Верхняя граница «нормального» диапазона (°C) |
| **`hot_slope`** | `0.05` | Прирост K_темп на каждый °C **выше** hot_threshold_c |

**Формула K_темп(T):**

```
T ≤ cool_threshold_c           →  cool_factor
cool_threshold_c < T ≤ hot_threshold_c  →  1.0
T > hot_threshold_c          →  1.0 + (T − hot_threshold_c) × hot_slope
```

Пример при defaults: 15°C → 0.8; 22°C → 1.0; 30°C → 1.0 + 3×0.05 = 1.15; 37°C → 1.5.

---

### `[model.rain]` — осадки за вчера (только уличные)

| Поле | По умолчанию | Смысл |
|------|--------------|-------|
| **`light_mm`** | `1.0` | Минимум осадков (мм), при котором срабатывает «лёгкий дождь» |
| **`heavy_mm`** | `10.0` | Выше этого порога — «сильный дождь» (уровень → 100%) |
| **`base_shift_hours`** | `24.0` | Базовый сдвиг полива при K_сезон = spring (эталон) |

**Правила** (осадки за вчера из Open-Meteo):

| Осадки | Эффект |
|--------|--------|
| **< light_mm** | Без изменений |
| **light_mm … heavy_mm** | Лёгкий дождь: статус `shifted_by_rain`, уровень растёт, полив сдвигается |
| **> heavy_mm** | Сильный дождь: уровень → 100%, статус `normal`, сброс отложений |

**Сдвиг полива при лёгком дожде** (связан с K_сезон — та же физика, что и высыхание):

```
K_эталон = spring                    (обычно 1.0)
factor   = K_эталон / K_сезон
         (зимой factor ограничен сверху: не больше 1.0)

сдвиг_часов = base_shift_hours × factor
```

Примеры при `base_shift_hours = 24` и defaults для season:

| Сезон | K_сезон | factor | Сдвиг |
|-------|---------|--------|-------|
| Лето | 1.5 | 1/1.5 | **16 ч** |
| Весна | 1.0 | 1/1.0 | **24 ч** |
| Осень | 0.75 | 1/0.75 | **32 ч** |
| Зима | 0.5 | min(2, 1) = **1.0** | **24 ч** |

Зимний потолок: при мёрзлой почве лёгкий дождь почти не удлиняет задержку.

**Добавление воды при лёгком дожде:**

```
добавлено = V_base × K_темп × (сдвиг_часов / 24)
уровень   = min(100, уровень + добавлено)
```

Здесь K_темп берётся от **текущей** температуры погоды, V_base = 100 / интервал_полива.

### Что **не** входит в `[model]`

| Поведение | Значение | Где задаётся |
|-----------|----------|--------------|
| CLI «**Уже полито**» (`--status`) | уровень ≥ **80%** | код `statuscli` (пока не параметризовано) |
| Критический полив | уровень ≤ **0%** | логика модели |
| Цвет полоски жажды на карточке | зелёный ≥ 80%, жёлтый 30–79%, красный < 30% | TUI (`moistureColor`), не связан с `water_soon_percent` |
| Текст `next_action` в TUI | не показывается | только БД / `--json` |

## Погода и дождь (уличные растения)

- Источник: **Open-Meteo** (`/v1/forecast`), город — в настройках.
- Обновление: раз в **N мин** (`weather_refresh_minutes` в `settings.toml`, **1–1440**, по умолчанию 10).
- В шапке: **город**, текущая температура, код погоды.
- Для автокорректировки берётся **сумма осадков за вчерашний день** (`daily.precipitation_sum`) в **таймзоне выбранного города**.
- Если вчерашний день в ответе API не найден — корректировка **не применяется** (до следующего успешного обновления).
- Если дневные данные недоступны, резерв: `current.precipitation` (только когда нет надёжных данных за вчера).
- Корректировка — **только уличным** растениям, **не чаще 1 раза в день** на город (ключ: дата + город + координаты; дата сохраняется в `settings.toml`).

### Правила по осадкам за вчера

| Осадки | Что происходит |
|--------|----------------|
| **< light_mm** (по умолчанию 1 мм) | Без изменений |
| **light_mm … heavy_mm** (по умолчанию 1–10 мм) | Статус `shifted_by_rain` 🌧; уровень воды **увеличивается**; полив откладывается на рассчитанное число часов |
| **> heavy_mm** (по умолчанию 10 мм) | Уровень → **100%**, статус `normal`, сброс отложений — «полито дождём» |

Пороги `light_mm` и `heavy_mm` настраиваются в `[model.rain]`. Подробные формулы — в разделе [Параметры модели полива](#параметры-модели-полива-model-в-settingstoml).

**Лёгкий дождь — расчёт** (см. также `[model.rain]` и `[model.season]`):

```
сдвиг_часов = base_shift_hours × (spring / K_сезон)    (зимой factor ≤ 1.0)
добавлено   = (100 / интервал) × K_темп × (сдвиг_часов / 24)
```

При значениях по умолчанию (`base_shift_hours = 24`, `spring = 1.0`):

| Сезон | K_сезон | Сдвиг |
|-------|---------|-------|
| Лето | 1.5 | **16 ч** (быстро сохнет — дождь откладывает меньше) |
| Весна | 1.0 | **24 ч** |
| Осень | 0.75 | **32 ч** (влага держится — дождь откладывает дольше) |
| Зима | 0.5 | **24 ч** (верхний предел: мёрзлая почва, лёгкий дождь почти не продлевает задержку) |

K_темп при добавлении воды — **текущая** температура из погоды.

На практике: в новый день, после первого успешного обновления погоды, приложение **один раз** применяет «вчерашний дождь» ко всем уличным растениям. На карточке отображается дата полива в формате **«Полив: ДД.ММ (дождь)»**.

## Эксплуатационные заметки

- Фоновая проверка жажды — **раз в минуту**; бар обновляется в памяти.
- Запись в БД — только при **заметном** изменении уровня/состояния или если прошло ≥ 30 мин с последней записи (меньше нагрузки на SQLite).
- Погодная корректировка и связанные записи в журнале — **не чаще одного раза в день** на выбранный город.
- При включённом **автобэкапе** при каждом запуске перезаписываются `data.db.bak` и `settings.toml.bak`.
- Ручной бэкап (`b` в настройках) создаёт консистентный снимок базы через `VACUUM INTO` и копирует `settings.toml`.

## Советы

- **Размер терминала.** Интерфейс подстраивается под ширину и высоту окна. При слишком маленьком терминале (< 40×10) появится предупреждение.
- **Терминал Kitty.** Иногда даёт мерцание или странное поведение стрелок в TUI. Если заметите глюки — попробуйте GNOME Terminal или Alacritty.
- **Подсказки внизу.** На каждой вкладке в футере показаны актуальные горячие клавиши.
- **Тайловые WM.** Используйте `--transparent` или настройку в приложении, чтобы не перекрывать свою тему терминала.
- **Тонкая настройка модели.** Коэффициенты высыхания, дождя и пороги — в `[model]` файла `settings.toml`; см. раздел [Параметры модели полива](#параметры-модели-полива-model-в-settingstoml).
- **Статус из shell.** `kiri --status` — таблица жажды и полива без запуска TUI (удобно для скриптов и быстрой проверки). `kiri --summary` — одна строка для виджетов и статус-бара. `kiri --json` — полный дамп растений в JSON.
